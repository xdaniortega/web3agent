# ERC-8004 Identity Registration

## What is ERC-8004?

ERC-8004 is an Identity Registry standard that gives onchain agents a verifiable identity. When an agent registers, it receives a unique agent ID that other agents and protocols can reference to verify the agent's identity and capabilities.

## Why register?

- **Discoverability** — other agents and dApps can find your agent by its ID
- **Verifiability** — the registration transaction is onchain and auditable
- **Interoperability** — builds toward a standard agent identity layer

## Registration flow

```
1. Agent wallet is created and funded
2. registerAgent() is called with name, description, privateKey, walletAddress
3. The @blockbyvlog/agent0-sdk submits a registration transaction
4. The agent receives an onchain agent ID
5. The agent can be viewed on 8004scan
```

## HTTP vs IPFS modes

### HTTP mode (default)

- No additional setup required
- Works out of the box on testnet
- Agent metadata is stored via HTTP

### IPFS mode (Pinata)

To use IPFS mode:

1. Get a [Pinata](https://www.pinata.cloud/) JWT
2. Set `PINATA_JWT` in your `.env`
3. In `src/registry.ts`, change `mode: "http"` to `mode: "ipfs"` and uncomment the `pinataJwt` line

IPFS mode stores agent metadata on IPFS via Pinata, providing decentralized and persistent storage.

## Usage

```ts
import { registerAgent } from "web3agent-sdk";

try {
  const result = await registerAgent({
    name: "my-agent",
    description: "Executes DeFi strategies on Arbitrum",
    privateKey: wallet.privateKey,
    walletAddress: wallet.address,
  });
  console.log(`Agent ID: ${result.agentId}`);
  console.log(`TX: ${result.txHash}`);
  console.log(`View: https://8004scan.com/agent/${result.agentId}`);
} catch (err) {
  console.error("Registration failed:", err);
}
```

## Verifying on 8004scan

After registration, view your agent at:

```
https://8004scan.com/agent/<agent-id>
```

## Known limitations

- The `@blockbyvlog/agent0-sdk` is in **alpha** — APIs may change
- Registration is best-effort and may fail under load or on certain networks
- Always wrap `registerAgent()` in try/catch
- The SDK's export structure may vary between versions — the registration module handles this with fallback import patterns
